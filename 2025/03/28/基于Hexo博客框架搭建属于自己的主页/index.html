基于Hexo博客框架搭建属于自己的主页

发表于2025-03-28|更新于2025-03-30|hexo搭建
|总字数:889|阅读时长:3分钟|浏览量:

常用的命令行指令

    在windows系统安装hexo框架不可避免地用到cmd命令行，下面是一些常用的指令：

  • 清空cmd命令行中的内容，cls
  • 在当前路径下建立文件夹，mkdir fileName
  • 查看当前命令行所在的文件路径，pwd（全称：print working directory）
  • 列出当前路径下有哪些内容，dir
  • 进入某个文件夹内 cd
  • 进入某个盘符，以E盘根目录为例 E：
  • 返回上一级 cd ..
  • 连续退回两层，即返回到上一级文件夹的再上一层，cd ../..

环境搭建

1、安装Nodejs
    下载Nodejs LTS版本，包括两个重要的组件一是Nodejs本身，二是npm包管理器

2、安装git

3、验证安装成功与否
    通过在命令行上运行node -v、npm -v以及git -v，查看nodejs、npm、git的版本，同时验证其是否安装成功

4、安装全局hexo框架
    npm install -g hexo-cli

github仓库建立并配置ssh key

1、新建gibhub仓库
    登录到自己的github，新建一个repository仓库，仓库名称必须按规则填写（用户名.github.io），且必须是public的，勾选Add a README file

2、配置ssh key
① 在桌面git bash here，输入命令ssh-keygen -t rsa -C“邮箱地址”，直接点击4下回车；
② 将自己电脑上C:\Users\用户名.ssh\id_rsa.pub的文字复制到github ——settings ——SSH and GPG keys——New SSH key中，名称随意；
③ 桌面上gitbash，ssh -T git@github.com，测试是否成功。

本地部署

1、在盘符根目录下新建一个文件用于存储相关文件

2、博客初始化
    gitbash here - hexo init （如果不行就npx hexo init）

3、安装
    hexo install

4、生成
​    hexo g

5、本地部署（会在本地localhost:4000地址上生成博客内容）
    hexo s

将自己的博客部署到github上公开使用

1、在3所创建的blog文件下，修改**_config.yml**里面的部署命令，注意缩进

2、hexo g生成

3、hexo d推到远端

4、配置邮箱和用户名

5、配置完成后，再次hexo d上传

写博客

hexo n “我的第一篇博客名称”

    在source/post文件下可以找到对应的文章.md文档，可以通过下载一个vim直接在命令行内操作，但是我还是更习惯通过hexo n 生成了md文档之后，再利用typora或者vscode编辑

清理，然后生成本地博客内容

或者
hexo cl;hexo s

更换主题及美化

    可以直接在b站上找其他人推荐的主题，然后在github看使用步骤就好了；
不过具体主题内容元素的配置参考别人配置学习
    一般，第一步就是git clone 通过ssh下载主题，二是下载需要的插件，三配置_config.yml

或者
hexo cl;hexo s

    这里特地补充：后续美化或者功能模块，可以在相关主题的配置官方文档上继续修改。
